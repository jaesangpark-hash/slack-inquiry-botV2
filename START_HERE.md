# 처음 받으신 분 — 여기부터 (이관·세팅 가이드)

> 받으신 것: 새 버전 봇 압축파일 1개(`slack-inquiry-bot-YYYYMMDD.tar.gz`).
> 이 문서는 **지금 운영 중인 기존 봇 코드를 안전하게 백업하고, 새 버전으로 갈아끼운 뒤, 봇을 켜기까지**를 안내합니다.

이 문서의 방식은 **여러분이 직접 명령어를 치지 않습니다.** 아래 `« »` 안의 **요청문을 복사해서, 쓰시는 AI 도구(파일을 읽고 컴퓨터 명령을 실행할 수 있는 AI)에게 그대로 붙여넣으면** AI가 백업·압축해제·설정 이전·실행을 대신합니다. 여러분은 **요청을 전달하고 결과를 확인**하면 됩니다.

> 봇을 켠 뒤의 평소 운영(켜기·끄기·문제 해결)은 `OPERATOR_GUIDE.md`,
> 코드를 직접 고치실 때는 `DEVELOPMENT.md`를 보세요. 이 문서는 **딱 한 번, 새 버전으로 갈아끼울 때** 보는 문서입니다.

---

## 0. 무엇이 바뀌나요? (1분 설명)

- **기능은 100% 그대로입니다.** 문의 분류·작품명 매칭·일정 파싱·권한 확인 등 동작은 지금과 똑같습니다.
- 바뀐 건 **코드를 어디에 두느냐(구조)** 뿐입니다. 예전엔 `app.js` 한 파일에 2,400줄이 몰려 있었는데, 이제 역할별 폴더(`src/...`)로 나뉘었습니다.
- 그래서 **부분 패치가 아니라 "새 코드 베이스 통째 교체"** 입니다. 기존 코드를 백업해 두고, 받으신 새 코드로 갈아끼웁니다.

> 비유: 책 내용(기능)은 그대로인데 챕터를 잘 나눠 목차를 단 새 판본입니다. 읽는 내용은 같고 찾기(수정)가 쉬워집니다.

---

## 1. 큰 흐름 (4단계)

```
[Step 1] 기존 코드를  legacy/ 폴더로 백업   (지금 쓰던 코드·설정 안전 보관)
[Step 2] 새 압축파일을 풀어서 작업 폴더에 전개   (새 구조: src/, package.json, .env.example ...)
[Step 3] 기존 설정값(.env·인증)을 새 .env로 이어받기   (대부분 그대로 재사용)
[Step 4] 설치하고 봇 켜기 + 정상 확인
```

각 단계는 아래에 **복사해서 AI에게 줄 요청문**으로 준비되어 있습니다.

---

## 2. ⚠️ 시작 전 — 가장 중요한 안전 수칙

1. **기존 `.env`와 인증 파일(예: `service-account.json`)은 절대 잃어버리면 안 됩니다.** 여기에 실제 토큰·열쇠가 들어 있습니다. Step 1에서 통째로 백업하므로, **백업을 먼저** 하고 그다음에 새 코드를 전개합니다.
2. **`.env`는 외부에 공유하거나 공개 저장소(GitHub 등)에 올리지 마세요.** (새 코드의 `.gitignore`에 이미 제외 설정이 되어 있습니다.)
3. 헷갈리면 AI에게 "지금 폴더 상태를 먼저 보여줘"라고 요청해 **현재 상태를 확인한 뒤** 진행하세요.

> AI에게 미리 일러두면 좋은 한마디:
> «앞으로 진행하는 동안, 내 기존 `.env` 파일과 인증 파일(service-account.json 등)은 절대 삭제하거나 덮어쓰지 말고, 반드시 백업본을 먼저 만들어 줘.»

---

## 3. 단계별 요청문

### Step 1 — 기존 코드를 `legacy/`로 백업

지금 봇이 돌아가고 있는 **작업 폴더 안에서** 진행합니다. AI에게:

> «지금 이 작업 폴더에 있는 기존 봇 코드 전체(`app.js`, `*Flow.js`, `findChannels.js`, `.env`, 인증 파일(`service-account.json` 등), `package.json`, 그 밖의 모든 파일)를 `legacy/`라는 새 하위 폴더를 만들어 통째로 옮겨 줘. **`.env`와 인증 파일도 반드시 포함**해서 옮기고, 하나도 빠뜨리지 마. 옮긴 뒤 `legacy/` 안의 파일 목록을 보여 줘.»

- 끝나면 작업 폴더에는 `legacy/` 폴더 하나만 남고, 그 안에 예전 코드가 전부 들어 있어야 합니다.
- `node_modules`(설치된 라이브러리 폴더)는 용량이 크면 굳이 안 옮겨도 됩니다. AI가 물어보면 "그건 안 옮겨도 돼"라고 답하세요.

### Step 2 — 새 압축파일 풀어서 작업 폴더에 전개

받으신 압축파일을 작업 폴더(또는 그 근처)에 둔 상태에서 AI에게:

> «내가 받은 새 버전 압축파일(`slack-inquiry-bot-*.tar.gz`)을 풀어 줘. 압축을 풀면 폴더가 하나 나올 텐데, **그 폴더 안의 내용물 전체를 지금 작업 폴더로 옮겨 줘** (`legacy/` 폴더는 그대로 두고). 그래서 작업 폴더에 `src/`, `package.json`, `.env.example`, `README.md` 등이 보이고, `legacy/`에는 예전 코드가 그대로 남아 있게 만들어 줘. 정리된 폴더 구조를 트리 형태로 보여 줘.»

- 끝나면 작업 폴더 = **새 코드(`src/` 구조)** + **`legacy/`(예전 코드 백업)** 구성이 됩니다.

### Step 3 — 기존 설정값(.env·인증)을 새 .env로 이어받기 ⭐

새 봇은 `.env` 파일에서 설정값 **23개**를 읽습니다. 대부분 **지금 쓰시던 값을 그대로 재사용**합니다. AI에게:

> «`.env.example`을 복사해서 새 `.env` 파일을 만들어 줘. 그다음 백업해 둔 `legacy/.env`에 들어 있던 내 기존 설정값들을, 새 `.env.example`의 항목과 **의미가 같은 것끼리 매칭**해서 새 `.env`에 채워 줘 (변수 이름이 약간 달라도 같은 의미면 매칭). 그리고 다음 두 가지를 처리해 줘:
> 1. **인증 정보**: 예전에 `service-account.json` 파일을 쓰고 있었다면, 그 파일 내용(JSON 전체)을 새 방식인 `GOOGLE_CREDENTIALS` 항목에 한 줄 문자열로 넣어 줘. 예전 `.env`에 이미 `GOOGLE_CREDENTIALS`가 있었다면 그대로 옮겨 줘.
> 2. **못 채운 항목**: 새로 생겼거나 기존 값에서 못 찾은 항목은 **빈 칸으로 두고, 그 목록을 따로 알려 줘** (내가 직접 값을 불러줄게).»

값을 알려달라고 하면 준비한 값을 불러 주면 됩니다. 다 채운 뒤 확인:

> «새 `.env`에 23개 항목이 빠짐없이 채워졌는지 확인하고, 비어 있는 항목이 있으면 이름을 알려 줘. 각 항목이 무슨 값인지 모르겠으면 `.env.example`의 한국어 주석을 참고해서 설명해 줘.»

> 23개 항목 = Slack 토큰·Gemini 키·Google 인증·시트 ID·채널 ID·게이트웨이 토큰·트리거 이모지명 등. 거의 다 지금 쓰시는 값이라 새로 발급할 건 거의 없습니다.

### Step 4 — 설치하고 봇 켜기 + 정상 확인

여기서부터는 평소 운영과 같습니다. 켜는 방법은 **Node.js** 또는 **Docker** 둘 중 편한 쪽입니다. 어느 게 나을지 AI에게 물어도 됩니다:

> «나는 개발 경험이 많지 않아. 이 봇을 Node.js로 켜는 것과 Docker로 켜는 것 중 어느 쪽이 더 간단할지 추천하고, 내 컴퓨터에 뭘 설치하면 되는지 알려 줘.»

**Node.js로 켜는 경우:**

> «이 폴더에서 봇에 필요한 라이브러리를 설치한 다음 봇을 실행해 줘. 실행 후 화면에 나오는 메시지를 알려 줘.»

**Docker로 켜는 경우 (Docker Desktop이 켜져 있어야 함):**

> «이 폴더에서 Docker로 봇을 빌드하고 백그라운드로 실행해 줘. 처음이라 시간이 좀 걸릴 수 있어. 진행 상황을 알려 줘.»

**정상 기동 확인:**

> «봇이 정상적으로 켜졌는지 로그를 확인해 줘. `🚀 시스템 가동! 준비 완료!` 메시지가 보이면 성공이라고 알려 줘. 에러가 있으면 그 내용을 쉬운 말로 설명해 주고, 가능하면 직접 고쳐 줘.»

마지막으로 **Slack에서 직접 확인**(사람만 가능):
1. 봇이 대상 채널에 멤버로 초대되어 있는지 확인.
2. 테스트 메시지에 **트리거 이모지**(`.env`의 `TRIGGER_EMOJI` 값, 예: `:문의봇소환:`)를 누릅니다.
3. 봇이 반응하면 정상입니다.

## 4. 한 번에 진행하고 싶다면 (통합 요청문)

단계를 하나씩 밟기보다 AI에게 한꺼번에 맡기고 싶다면 아래를 복사해 주세요. (그래도 **각 단계 결과는 꼭 확인**하세요.)

> «지금 이 작업 폴더에는 운영 중인 Slack 봇 코드가 있고, 나는 같은 봇의 새 버전 압축파일(`slack-inquiry-bot-*.tar.gz`)을 받았어. 다음 순서로 진행해 줘. 각 단계가 끝날 때마다 결과를 보여주고 다음으로 넘어가:
> 1. 기존 코드 전체(`.env`·인증 파일 포함)를 `legacy/` 폴더로 백업. 절대 삭제·덮어쓰기 금지.
> 2. 새 압축파일을 풀어 그 내용물을 이 작업 폴더로 전개(`legacy/`는 보존). 결과 폴더 구조를 보여줘.
> 3. `.env.example`을 바탕으로 새 `.env`를 만들고, `legacy/.env`의 값과 `service-account.json`(있으면 그 JSON을 `GOOGLE_CREDENTIALS`로) 내용을 의미에 맞게 옮겨. 못 채운 항목은 빈 칸으로 두고 목록을 알려줘.
> 4. 빈 항목을 내가 채운 뒤, 봇을 실행하고 `🚀 시스템 가동! 준비 완료!` 로그가 뜨는지 확인해줘.
> 진행 중 내 기존 `.env`·인증 파일은 반드시 백업본을 유지하고, 작업 폴더 밖으로는 어떤 비밀값도 내보내지 마.»

## 5. 문제가 생겼거나 되돌리고 싶을 때

**증상: 켜자마자 멈추고 `필수 환경변수 누락` 같은 메시지가 뜬다**

> «봇을 켰더니 이런 메시지가 나오면서 멈췄어: [메시지 전체 붙여넣기]. 원인이 뭐고 어떻게 고치면 되는지 쉬운 말로 알려 주고, 가능하면 직접 고쳐 줘.»

(대개 `.env`에 빈 칸이 있다는 뜻이며, 메시지 끝에 빠진 항목 이름이 표시됩니다 → Step 3으로 돌아가 채우기.)

**예전 버전으로 되돌리고 싶다 (롤백)**

새 버전에 문제가 있어 일단 예전 봇으로 돌아가려면:

> «새 버전을 멈추고, `legacy/` 폴더에 백업해 둔 예전 코드로 봇을 다시 켜 줘. 예전 코드는 `legacy/` 안에 그대로 있으니 그걸 기준으로 실행하면 돼.»

백업을 Step 1에서 해 두었으므로 **언제든 예전 상태로 복구**할 수 있습니다.
