# 운영 가이드 (비개발자용 · AI 어시스턴트 활용)

> 🔰 **새 버전을 처음 받으셨다면 먼저 `START_HERE.md`로 이관·세팅(기존 코드 백업 → 새 코드 교체 → 설정 이어받기)을 끝낸 뒤** 이 문서로 평소 운영을 진행하세요.

이 문서는 **개발 경험이 없는 분**이 **AI 어시스턴트의 도움을 받아** 이 봇을 켜고 운영하도록 안내합니다.

핵심 방식: **여러분이 직접 명령어를 입력하지 않습니다.** 대신 아래에 준비된 **요청문을 복사해 AI에게 건네면**,
AI가 명령 실행·파일 편집·로그 확인·문제 진단을 대신합니다. 여러분은 **요청을 전달하고 결과를 확인**하면 됩니다.

> 요청문은 모두 `« »` 안에 있습니다. 그대로 복사해서 AI에게 붙여넣으세요.
> 기술 배경이 궁금하면 `README.md`, 코드를 직접 고치려면 `DEVELOPMENT.md`를 참고하세요.

---

## 0. 이 봇이 하는 일 (1분 설명)

- Slack 채널에서 작업자가 남긴 **문의 메시지**에 봇이 **자동으로 응답**합니다.
- 메시지에 **특정 이모지(예: `:문의봇소환:`)를 누르면** 봇이 그 문의를 읽고,
  일정·납품·재수급·리테이크 등 종류를 구분해 필요한 정보를 찾아 답합니다.
- 작품명·일정은 **Google 시트**에서, 권한 확인은 **코드에 등록된 허용 목록**에서, 문장 이해는 **AI(Gemini)** 가 담당합니다.
- 허용 담당자는 코드 목록(`permission-gate.js`)에 있으며, 추가·제외는 AI에게 요청 가능합니다.

봇이 동작하려면 이 서비스들에 접속할 **열쇠(토큰·ID)** 가 필요합니다. 그게 아래 "설정값"입니다.

## 1. 시작 전 준비

### 1-1. AI 어시스턴트 준비

- **파일을 읽고 컴퓨터 명령을 실행할 수 있는 AI 어시스턴트**(코드 작업이 가능한 AI 도구)를 준비하세요.
- 그 AI에게 **이 폴더(프로젝트 전체)를 열어 주거나 위치를 알려** 주세요.
- 이후 단계에서는 아래 요청문을 그 AI에게 전달합니다.

### 1-2. AI가 대신 해주는 일 vs 직접 해야 하는 일

| AI가 대신 해줌 (요청문 전달) | 직접 해야 함 (AI에게 방법은 물어볼 수 있음) |
|---|---|
| `.env` 설정 파일 만들고 값 채우기 | 토큰·시트 ID 등 **설정값을 발급·확보** (웹 콘솔에서) |
| 봇 빌드·실행·중지 | 필요한 프로그램(Docker 또는 Node.js) **설치 클릭** |
| 로그 확인·정상 기동 확인 | 봇을 **Slack 채널에 초대**, 메시지에 **이모지 클릭** |
| 에러 원인 진단·해결책 제시 | 봇을 켜둘 **컴퓨터를 계속 켜두기** |

### 1-3. 설정값 23개 (미리 확보)

대부분 **이미 운영하던 값을 그대로 재사용**할 수 있습니다(Slack 토큰·Gemini 키·Google 인증·시트 ID·채널 ID·Totus 토큰·트리거 이모지명 등).
각 값이 무엇이고 어디서 얻는지 모르겠다면 AI에게:

> «이 폴더의 `.env.example` 파일을 열어서, 각 설정값이 무엇이고 어디서 얻는지 한국어로 쉽게 설명해 줘.»

## 2. 실행 방법 정하기

봇을 켜는 방법은 **Docker** 또는 **Node.js** 두 가지입니다. 어느 게 나을지 AI에게 물어보세요:

> «나는 개발 경험이 없어. 이 봇을 Docker로 켜는 것과 Node.js로 켜는 것 중 어느 쪽이 더 간단할지 추천하고, 내 컴퓨터에 무엇을 설치하면 되는지 알려 줘.»

설치 자체(데스크톱 앱 설치 등)는 직접 클릭해야 하지만, **단계는 AI가 안내**해 줍니다:

> «(Docker 또는 Node.js)를 내 컴퓨터에 설치하는 방법을 단계별로 알려 줘. 설치가 끝났는지 확인하는 방법도 알려 줘.»

---

## 3. 설정 파일(`.env`) 만들기 — AI에게 요청

봇은 `.env` 파일에서 설정값을 읽습니다. AI에게 이렇게 요청하세요:

> «이 폴더의 `.env.example` 파일을 복사해서 `.env` 파일을 만들어 줘. 그다음 항목을 하나씩 물어봐 주면 내가 값을 불러줄게. 내가 부른 값을 해당 항목에 채워서 저장해 줘. 23개 항목을 모두 채울 때까지 진행하자.»

- AI가 항목을 하나씩 물으면, 준비한 값을 불러 주면 됩니다.
- 다 채운 뒤 확인 요청:

> «`.env`에 23개 값이 빠짐없이 채워졌는지 확인하고, 빈 항목이 있으면 알려 줘.»

> ⚠️ **주의**: `.env`에는 비밀 열쇠가 들어갑니다. **외부에 공유하거나 공개 저장소에 올리지 마세요.** (AI에게 "절대 외부로 보내지 말라"고 일러둬도 좋습니다.)

---

## 4. 봇 켜기 — AI에게 요청

설정 파일이 준비되면, 정한 방법에 따라 AI에게 요청하세요.

**Docker로 켜는 경우** (Docker Desktop이 실행 중이어야 함):

> «이 폴더에서 Docker로 봇을 빌드하고 백그라운드로 실행해 줘. 처음이라 시간이 좀 걸릴 수 있어. 실행 명령과 진행 상황을 알려 줘.»

**Node.js로 켜는 경우**:

> «이 폴더에서 봇 의존성을 설치한 다음 봇을 실행해 줘. 실행 후 화면에 나오는 메시지를 알려 줘.»

---

## 5. 잘 켜졌는지 확인 — AI에게 요청

> «봇이 정상적으로 켜졌는지 로그를 확인해 줘. `🚀 시스템 가동! 준비 완료!` 메시지가 보이면 성공이라고 알려 줘. 에러가 있으면 그 내용을 쉬운 말로 설명해 줘.»

그다음 **Slack에서 직접 확인**합니다(이건 사람만 할 수 있습니다):

1. 봇이 **대상 채널에 멤버로 초대**되어 있는지 확인(초대 안 되면 메시지를 못 봄).
2. 테스트 메시지에 **트리거 이모지**(`.env`의 `TRIGGER_EMOJI`에 넣은 이름, 예: `:문의봇소환:`)를 누릅니다.
3. 봇이 응답하면 정상입니다.

> 트리거 이모지는 **Slack에 미리 등록된 커스텀 이모지**여야 하고, 봇 앱에 `reactions:read`/`reactions:write` 권한이 있어야 합니다. 설정이 안 되어 있으면 AI에게 «봇이 Slack에서 이모지에 반응하려면 어떤 설정이 필요한지 알려 줘.» 라고 물어보세요.

---

## 6. 끄기 / 다시 켜기 — AI에게 요청

> «봇을 안전하게 중지해 줘.»

> «봇을 다시 켜 줘. (설정이나 코드를 바꿨다면 새로 빌드해서 켜 줘.)»

---

## 7. 문제가 생겼을 때 — AI에게 진단 요청

문제가 보이면 **그 내용을 그대로 복사해 AI에게** 전달하세요.

**증상: 켜자마자 멈추고 `필수 환경변수 누락` 같은 메시지가 나온다**

> «봇을 켰더니 이런 메시지가 나오면서 멈췄어: [여기에 메시지 전체를 붙여넣기]. 원인이 뭐고 어떻게 고치면 되는지 쉬운 말로 알려 주고, 가능하면 직접 고쳐 줘.»

(보통 `.env`에 빈 칸이 있다는 뜻이며, 메시지 끝에 빠진 항목 이름이 표시됩니다.)

**증상: 봇이 Slack 메시지에 반응하지 않는다**

> «봇이 Slack에서 이모지에 반응하지 않아. 최근 로그를 확인하고 원인을 진단해 줘. 채널 초대, 이모지 이름, 토큰 만료 가능성을 차례로 점검해 줘.»

**증상: 봇이 「⚠️ … 결과가 불명확해 운영자 확인이 필요해. … 버튼을 다시 누르지 말아줘.」 라고 안내한다**

이건 고장이 아니라 **일부러 만든 안전장치**입니다. 시트 기록·메시지 전송·파일 업로드 같은 작업이 **서버에는 반영됐는데 네트워크 응답만 끊긴** 상황일 수 있어, 봇이 **자동으로 다시 실행하지 않고 멈춘** 것입니다.

- **같은 버튼을 다시 누르지 마세요.** 다시 누르면 중복(시트 행 2개·메시지 2개·파일 2개)이 생길 수 있습니다.
- 대신 **실제로 반영됐는지 직접 확인**하세요 — 해당 Google 시트에 행이 생겼는지, Slack 스레드에 메시지가 올라갔는지, Totus(플랫폼)에 반영됐는지.
- 판단이 어려우면 AI에게:

> «봇이 "[안내 메시지 전체를 붙여넣기]"라고 떴어. 실제로 반영됐는지 확인하는 방법을 알려 주고, 로그로 마지막 작업 상태를 점검해 줘. 버튼은 다시 누르지 않을게.»

**증상: 봇을 껐다 켠(재시작) 직후가 걱정된다**

봇은 진행 중이던 작업 상태를 주로 메모리에 두기 때문에, **게시나 파일 업로드가 진행되던 바로 그 순간에 봇이 재시작**되면 드물게 **시트 행이나 파일이 한 번 중복**될 수 있습니다(문의 완료·작업자 안내 경로는 스레드 표식으로 중복을 막습니다). 재시작 직후에는 **마지막으로 처리하던 작업이 실제로 어떻게 반영됐는지 한 번 확인**하세요.

**도움을 사람(개발자)에게 요청할 때**: 아래처럼 로그를 정리해 달라고 한 뒤, 받은 내용을 전달하세요.

> «개발자에게 도움을 요청하려고 해. 최근 로그의 핵심 부분과 지금 상황을 한 단락으로 정리해 줘.»

## 8. 요청문 모음 (빠른 참조)

| 하고 싶은 것 | AI에게 보낼 요청문 |
|---|---|
| 설정값 이해 | «`.env.example`의 각 값이 무엇이고 어디서 얻는지 쉽게 설명해 줘.» |
| 설정 파일 만들기 | «`.env.example`을 `.env`로 만들고, 항목을 하나씩 물어봐서 내가 부른 값으로 채워 줘.» |
| 봇 켜기 (Docker) | «이 폴더에서 Docker로 봇을 빌드하고 백그라운드로 실행해 줘.» |
| 봇 켜기 (Node.js) | «이 폴더에서 의존성을 설치하고 봇을 실행해 줘.» |
| 상태 확인 | «로그를 확인해서 `🚀 시스템 가동! 준비 완료!`가 떴는지 알려 줘.» |
| 봇 끄기 | «봇을 안전하게 중지해 줘.» |
| 설정 바꾸고 재시작 | «`.env`의 (항목)을 (값)으로 바꾸고 봇을 다시 켜 줘.» |
| 문제 진단 | «봇에서 이런 에러가 났어: [붙여넣기]. 원인과 해결법을 알려 주고 고쳐 줘.» |
| "확인 필요" 안내가 떴을 때 | «봇이 "[안내 메시지]"라고 떴어. 실제 반영 여부 확인 방법을 알려 주고 로그로 점검해 줘. 버튼은 다시 누르지 않을게.» (⚠️ 같은 버튼 재클릭 금지 — 중복 위험) |

이 가이드의 모든 단계는 **AI에게 요청문을 전달하고 결과를 확인하는 방식**으로 진행됩니다.
설치 클릭·설정값 확보·Slack 초대·이모지 클릭처럼 사람만 할 수 있는 일도 **방법은 AI에게 물어** 진행하면 됩니다.
